# 智能招聘 Agent — 产品设计规划

> 版本: v2.1 | 日期: 2026-07-07 | 状态: 6项交互优化完成（主对话框改造/R-P-R-A-R流式/任务详情页/简历补录/标签输入/权重滑块）
> **UI 文件目录:** `ui/` — 8 个 MVP 页面 + 1 个任务详情页 + 1 个 Phase 2 占位页，与本文档章节一一对应。修改任一页面后请同步更新本文档。
> **导航结构:** 12 项全局导航（核心流程 6 + 分析与设置 2 + 更多功能/Phase 2 共 4 项缺省页）
> **首页:** 任务看板（page-2）为首页入口，新建任务进入任务中心（page-1）对话界面

---

## 一、产品定位与核心架构

### 1.1 产品定位

面向企业 HR 的任务驱动型智能招聘 Agent，通过自然语言交互覆盖「JD 准备 → 简历筛选 → 反馈闭环」核心链路。

### 1.2 核心目标

- 招聘事务性工作时间降低 50%
- Top 20% 候选人与人工重合率 > 70%
- MVP 8 周交付，20+ 种子用户内测验证

### 1.3 核心架构决策

| 维度 | 决策 | 说明 |
|------|------|------|
| Agent 推理框架 | R-P-R-A-R 思考链 | Reason → Reflect → Plan → Reflect → Act → Reflect；Plan 阶段为关键创新点，执行前生成步骤清单，用户确认后执行 |
| 记忆架构 | 四层记忆 | L1 工作记忆(Redis)、L2 任务记忆(PG)、L3 实体记忆(PG+向量)、L4 组织记忆(PG+对象存储)；MVP 阶段 L2/L3/L4 共用 PG 不同表 |
| 前端 | React 18 + Ant Design 5 + TypeScript | 对话区域自定义消息流组件，支持 SSE 流式渲染 |
| 后端 | Python FastAPI + LangGraph | LangGraph 编排 Agent 状态机 |
| LLM | 多模型 API 适配层 | 支持通过后台配置接入任意兼容 OpenAI API 格式的大模型（GPT-4 / Claude / DeepSeek / 通义千问 / 本地 Ollama 等），可在系统设置页面配置 API Key、Base URL、模型名称，支持运行时切换和 A/B 测试 |
| 数据库 | PostgreSQL 15 + Redis 7 | PG 支持 JSONB 和全文检索 |
| 文件存储 | MinIO | S3 兼容，私有化部署 |
| 监控 | Prometheus + Grafana | 基础看板 |

### 1.4 交付范围

- **MVP 阶段（完整 UI 设计）**：5 大核心模块 + 基础版反馈学习 + 基础看板
- **Phase 2（概要框架）**：面试安排、候选人沟通、数据分析增强

## 二、核心设计理念

### 2.1 对话驱动的任务中心

产品核心交互不是传统表单，而是对话式交互。Agent 通过 R-P-R-A-R 思考链完成推理，用户通过自然语言下达指令，流式输出让用户实时感知 Agent 的"思考过程"。

### 2.2 可控性与信任感

- Plan 阶段展示执行步骤清单，用户可确认/调整/跳过
- 评分维度和权重可实时调整，预览排序影响
- 关键操作（推送、发布 JD）前强制确认
- 每个决策都可回溯查看思考链日志

### 2.3 双向关联数据模型

JD 与候选人之间建立双向关联关系：
- JD → 候选人：每个 JD 可查看其历史筛选的所有候选人及状态
- 候选人 → JD：每个候选人可查看其被哪些 JD 筛选过及对应评分
- 评分记录同时记录 `jd_id` 和 `candidate_id`

## 三、MVP 页面结构

### 页面 1：对话式任务中心（主界面）

> **UI 原型:** `ui/page-1-chat-task-center.html`
> **详情页:** `ui/page-1b-task-detail.html`

**定位：** 产品的核心入口，承载 Agent 自然语言交互和任务管理。统一主对话框，通过顶部岗位选择器切换不同招聘任务。

**布局：**
- **左侧边栏（240px）：**
  - 全局导航（12项）
  - 底部用户信息
- **顶部岗位选择器：**
  - 下拉框选择当前操作的岗位（后端开发工程师、高级产品经理等）
  - 显示当前阶段 badge（如"简历筛选中"）
  - 切换时在对话区插入系统提示消息
- **右侧主区域 — R-P-R-A-R 流式对话流：**
  - Reason（推理）：灰色思考气泡，展示 Agent 分析过程
  - Reflect（反思）：浅蓝气泡，展示自我审视
  - Plan（计划）：可交互步骤卡片，含预估时间（确认/调整/跳过）
  - Reflect-Plan（计划反思）：浅蓝气泡，评估计划合理性
  - Tool Call（工具调用）：可折叠卡片，显示工具名/状态/参数
  - Progress（进度）：进度条 + 实时计数
  - Result（结果）：完整结果卡片
  - Reflect-Act（执行反思）：浅蓝气泡，总结执行效果
- **底部输入区：**
  - 文本输入框 + 发送按钮
  - 快捷功能标签：生成JD / 筛选简历 / 查看报告 / 任务切换（点击填入意图文本，不跳转页面）
  - 文件上传按钮（简历拖拽上传）

### 页面 1b：任务详情（岗位招聘全过程）

> **UI 原型:** `ui/page-1b-task-detail.html`

**定位：** 单个岗位招聘任务的完整过程视图，从任务看板或任务中心进入。

**布局：**
- **头部：** 返回按钮 + 面包屑 + 岗位名称 + 状态标签 + meta 信息（创建时间/负责人/部门/简历数）+ 操作按钮（进入对话/查看JD）
- **进展指示器：** 5 阶段横向步骤条（需求解析 → JD编写 → 简历筛选 → 评分推送 → 已归档），已完成绿色勾，当前高亮
- **Tab 1 — 会话记录：** 时间线形式的对话历史（时间戳 + 角色 + 摘要）
- **Tab 2 — 关联资料：** 关联JD卡片 + 候选人简历列表（姓名/匹配分/状态/操作）
- **Tab 3 — 操作日志：** 时间线日志（时间 + 操作类型图标 + 描述 + 操作人）

### 页面 2：任务管理看板

> **UI 原型:** `ui/page-2-task-kanban.html`

**定位：** 所有招聘任务的全局视图，支持看板和列表两种模式。

**布局：**
- 顶部：筛选栏（按部门/状态/时间范围）
- 看板视图：6 列泳道
  - 需求解析中 → JD 编写中 → 简历筛选中 → 待推送 → 已推送 → 已归档
- 任务卡片信息：职位名称、创建时间、当前阶段、简历数量、负责人、状态标签
- 操作：创建任务、编辑、归档、批量操作

### 页面 3：JD 管理页面

> **UI 原型:** `ui/page-3-jd-management.html`

**定位：** JD 模板库管理和编辑中心。

**布局（三 tab 结构）：**

**Tab 1：JD 模板库**
- 标准 JD 模板列表（支持搜索、按职位类别筛选）
- 每个模板卡片：职位类别、标题、版本号、使用次数、创建时间
- 操作：新建模板、编辑、复制、版本历史

**Tab 2：JD 编辑器**
- 双模式编辑：字段化结构表单 + 富文本编辑器
- 左侧结构化字段：职位名称、部门、职级、技能要求、经验年限、学历、薪资范围、职责描述
- 右侧预览区：实时渲染 JD 最终效果
- 合规校验结果面板（高亮问题字段，如歧视性词汇）
- 版本快照按钮：保存当前版本
- 发布按钮：提交合规校验后发布

**Tab 3：关联候选人（JD → 候选人方向）**
- 当前 JD 历史筛选过的所有候选人列表
- 每个候选人：姓名、评分、状态（已推送/已反馈/已淘汰）、筛选时间
- 操作：查看详情、发起新一轮筛选（复用此 JD + 上传新简历）
- 筛选条件：按评分范围、状态筛选

### 页面 4：简历工作台

> **UI 原型:** `ui/page-4-resume-workspace.html`

**定位：** 简历上传、解析、查看的核心工作区。

**布局（三区域）：**

**顶部：上传区**
- 拖拽上传区域（支持文件夹/ZIP 包）
- 格式支持标识：PDF / DOCX / JPG-PNG(OCR)
- 单次上限 100 份，每份 < 10MB
- 上传进度展示

**中部：解析结果列表**
- 列表视图，每行一个候选人
- 状态标签：✅ 解析成功 / ⚠️ 低置信度 / ❌ 解析失败
- 可展开行：展示结构化提取的 5 章节详情
  - 基本信息（姓名、联系方式）
  - 工作经历（公司、职位、时间、职责）
  - 教育背景（学校、专业、学历、时间）
  - 技能列表
  - 项目经验
- 操作：查看详情、人工补录、重新解析

**右侧面板：关联 JD（候选人 → JD 方向）**
- 选中某个候选人后，右侧显示其关联 JD 列表
- 每个 JD：职位名称、评分、筛选时间、筛选任务
- 点击可跳转到该 JD 的关联候选人视图

**底部：失败简历处理区**
- 解析失败的简历单独列出
- 失败原因分类：文件损坏 / 密码保护 / 格式不支持 / 内容为空
- 操作：重新解析 / 人工补录入口

### 页面 5：评分与筛选报告页面

> **UI 原型:** `ui/page-5-scoring-report.html`

**定位：** 评分配置、候选人排名、推荐决策的核心页面。

**布局（三栏）：**

**左栏（280px）：评分维度配置面板**
- 5 个评分维度滑块：
  - 技能匹配（默认 35%）
  - 经验年限（默认 25%）
  - 项目相关性（默认 20%）
  - 教育背景（默认 10%）
  - 稳定性（默认 5%）
  - 其他/预留（默认 5%）
- 权重百分比实时显示，总和强制 = 100%
- "保存为默认"按钮
- 实时预览：调整权重后，右侧排序列表即时更新

**中栏（主区域）：候选人排序列表**
- 排序方式：总分降序（默认）、可按各维度单独排序
- 每个候选人卡片：
  - 排名、姓名、总分（大号数字）
  - 维度雷达图（小型，5 维）
  - 亮点标签（绿色）：如"技术栈高度匹配""大厂背景"
  - 风险标签（橙色）：如"跳槽频繁""经验年限不足"
  - 推荐理由（一句话）
  - 操作按钮：查看详情、标记推荐、标记淘汰
- 批量操作栏：全选、批量推荐、批量淘汰、导出报告

**右栏（可展开/收起）：候选人详情面板**
- 简历原文预览
- 结构化字段摘要
- 评分 evidence 详情（每个维度的得分/满分 + 证据文本）
- 关联 JD 信息
- 反馈记录（如有）

### 页面 6：推送与反馈页面

> **UI 原型:** `ui/page-6-push-feedback.html`

**定位：** 结果推送和业务反馈闭环。

**布局（两 tab）：**

**Tab 1：推送管理**
- 推送预览卡片：
  - 候选人列表（头像/姓名/评分/推荐理由摘要）
  - 推送目标选择（业务经理下拉选择）
  - 附加信息：JD 链接、评分维度配置摘要
- 推送历史列表：
  - 推送时间、目标人、候选人数量、状态（已读/未读/已反馈）
- 操作：确认推送、撤回

**Tab 2：反馈收集**
- 业务经理视图的反馈面板
- 每个候选人一行：
  - 候选人姓名 + 评分 + 简历摘要
  - 三选一按钮：符合预期 / 部分符合 / 不符合
  - 选择"部分符合"或"不符合"时展开原因多选：
    - 技术不匹配 / 经验不足 / 沟通问题 / 薪资超预期 / 其他（可填写）
- 反馈统计概览（底部）：
  - 符合/部分符合/不符合 饼图
  - 常见拒绝原因排行

### 页面 7：基础数据看板

> **UI 原型:** `ui/page-7-analytics-dashboard.html`

**定位：** 系统运行健康度和用户接受度监控。

**布局（两 tab）：**

**Tab 1：功能健康度**
- 意图识别准确率（日/周趋势折线图）
- 各工具调用成功率（柱状图）与延迟 P95（折线图）

**Tab 2：用户接受度**
- DAU（日活趋势折线图）
- 人均任务完成数
- 人工接管比例（日趋势）
- 用户覆盖 Agent 决策的位置分布（热力图）
- NPS 评分（周统计）

### 页面 8：系统设置

> **UI 原型:** `ui/page-8-system-settings.html`

**定位：** 管理员配置中心，包括 LLM 接入、评分引擎参数、系统参数等。

**布局（多 tab 结构）：**

**Tab 1：LLM 模型配置**
- 已配置模型列表（名称、提供商、Base URL、当前模型标识、状态）
- 新增模型配置表单：
  - 配置名称（如"GPT-4o 生产环境"）
  - API Base URL
  - API Key（密文存储，输入时 mask 显示）
  - 模型名称（如 gpt-4o、deepseek-chat、qwen-plus）
  - 最大 Token 数
  - 温度参数（默认值）
- 操作：设为默认模型、启用/禁用、测试连通性、删除
- 支持配置多个模型实例，按任务类型分配不同模型（如意图识别用轻量模型、JD 生成用高质量模型）

**Tab 2：评分引擎参数**
- 默认评分维度配置（维度名称、默认权重、启用/禁用）
- 自定义维度管理（新增/编辑/删除企业自定义维度）
- 评分阈值配置（低置信度阈值、强制人工复核阈值）

**Tab 3：系统参数**
- 简历解析批量上限
- 追问轮次上限
- Token 消耗预算（日/月额度）
- 数据保留策略（简历保留期限、任务归档周期）

## 四、Phase 2 概要框架

### 模块 8：面试安排

**流程框架：**
1. 从筛选报告确认的面试名单 → 查询面试官日历空闲时段
2. 推荐面试时间段（考虑候选人偏好 + 面试官空闲 + 会议室）
3. 发送面试邀请（邮件/企微通知）
4. 候选人响应状态机（接受/改期/拒绝）
5. 改期处理：重新查询时段 → 重新推荐

**页面概念：**
- 日历集成视图（面试官日历 + 面试时段标记）
- 面试列表管理（时间/候选人/面试官/状态）
- 面试详情页（候选人背景摘要 + 面试反馈录入）

### 模块 9：候选人沟通

**流程框架：**
1. 沟通模板库管理（通知模板、进度同步模板、FAQ 模板）
2. 触发式沟通（面试邀请/结果通知/进度更新）
3. 沟通记录时间线（与候选人所有交互的按时间排列）

**页面概念：**
- 沟通模板库列表 + 编辑器
- 候选人沟通时间线视图
- 批量沟通发送界面

### 模块 10：数据分析增强

**流程框架：**
- 三级埋点体系全面落地（行为埋点 + 任务埋点 + 模型埋点）
- 四看板升级（实时监控 + 效果评估 + 用户行为 + 模型性能）
- A/B 测试框架（评分模型版本切换、灰度发布）

**页面概念：**
- 模型性能看板（各模型准确率/延迟、Prompt A/B 测试、Bad-case 聚类）
- 用户行为看板（功能使用热力图、决策覆盖位置分布、任务漏斗转化）
- A/B 测试管理界面

## 五、设计风格规范

### 5.1 配色方案

- **主色：** Teal (#0D9488) — 专业、可信赖
- **强调色：** Orange (#EA580C) — 警告、需注意的元素
- **成功色：** Green (#16A34A) — 正向反馈、通过
- **背景色：** Warm Gray (#FAFAF9 / #F5F5F4) — 柔和舒适的底色
- **文字色：** Stone (#292524) — 主要文字
- **辅助文字：** Stone (#78716C) — 次要说明

### 5.2 字体

- **标题字体：** Outfit — 现代、清晰
- **正文字体：** WorkSans — 专业、易读
- **代码字体：** JetBrainsMono — 技术信息展示

### 5.3 设计原则

- 信息密度优先（B 端效率工具）
- 操作路径最短化（减少点击层级）
- 状态可视化（颜色标签 + 图标 + 进度指示）
- 异常醒目但不干扰（warning 条/而非弹窗）

## 六、流式消息 UI 设计规范

### 6.1 七种消息类型

| 消息类型 | 展示样式 | 用户可交互 |
|----------|----------|------------|
| `thinking` | 灰色小字 + loading 动画 | 不可交互，可跳过 |
| `plan` | 结构化任务清单卡片（步骤列表 + 预估耗时 + 确认点） | 可确认/调整/跳过步骤 |
| `tool_call` | 折叠卡片（工具名 + 调用原因） | 可展开查看参数 |
| `progress` | 进度条 + 实时计数 | 不可交互，可取消 |
| `question` | 高亮气泡 + 选项按钮/输入框 | 必须交互才能继续 |
| `result` | 完整卡片/表格/报告 + 操作按钮 | 可操作（确认/修改/重新执行） |
| `warning` | 橙色警告条 + 问题说明 + 建议 | 可选择继续或人工接管 |

## 七、非功能需求

| 类别 | 需求 | 验收标准 |
|------|------|----------|
| 性能 | 简历解析吞吐量 | 单节点 > 50 份/分钟 |
